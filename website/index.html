Why Fajar Lang?

Built for engineers who need ML inference and OS control in a single binary.

Embedded ML

First-class Tensor type with shape checking at compile time. Autograd, Conv2d, Attention, INT8 quantization -- all native to the language.

OS Integration

Write bare-metal kernels in the same language. IRQ handlers, page tables, syscalls, inline assembly -- with compiler-enforced context isolation.

Safety by Construction

Ownership-based memory safety without lifetime annotations. @kernel and @device contexts prevent category errors at compile time.

Performance

Cranelift + LLVM native codegen with ARM64 and RISC-V cross-compilation. No garbage collector, no runtime overhead, deterministic execution.

Async/Await

Real async I/O via tokio — async_sleep, async_http_get, async_spawn, async_join, async_select. Cooperative + real concurrency.

Networking

WebSocket (tungstenite + TLS), MQTT (rumqttc), BLE (btleplug), HTTP/HTTPS server framework with router + middleware.

Formal Verification

SMT-backed safety proofs for @kernel and @device code. Symbolic execution, property annotations, DO-178C/ISO 26262 certification support.

Distributed Runtime

Raft consensus, service discovery, data-parallel ML training, ring-allreduce. Deploy across clusters with fault tolerance and mTLS security.

WASI P2 Components

Full WebAssembly Component Model — WIT parser, canonical ABI, HTTP client/server, filesystem, sockets. Deploy to wasmtime, Spin, or any WASI runtime.

FFI v2 (C++/Python/Rust)

Call C++ templates and STL, Python async and NumPy/PyTorch, Rust traits and futures — all with automatic binding generation via fj bindgen.

Self-Hosting Compiler

The compiler can compile itself. Tree AST, Pratt parser, HM type inference, 46-opcode bytecode codegen, Stage 2 bootstrap verified.

Developer Tools

LSP with 18 features, VS Code extension, formatter, debugger (DAP), test framework, package manager, playground.

Real Threaded Actors

Spawn lightweight actors on real OS threads with std::thread + mpsc channels. Supervision strategies, message passing, and fault isolation built in.

LLVM JIT & AOT Backend

Full LLVM backend alongside Cranelift -- JIT for rapid prototyping, AOT for production ELF/Mach-O binaries with link-time optimization.

48 Production Builtins

48 built-in functions verified end-to-end with zero simulated stubs. From tensor ops to OS primitives, every builtin works in production.

See It in Action

// Bridge OS sensor data to ML inference -- safely

@kernel
fn read_sensor() -> [f32; 4] {
    let base: ptr = 0x4000_1000
    [port_read(base), port_read(base + 4),
     port_read(base + 8), port_read(base + 12)]
}

@device
fn classify(input: Tensor) -> Tensor {
    let model = Dense(4, 3) |> relu |> softmax
    model.forward(input)
}

@safe
fn main() {
    let raw = read_sensor()
    let prediction = classify(Tensor::from_slice(raw))
    match prediction.argmax() {
        0 => println("Normal"),
        1 => println("Warning"),
        _ => println("Critical"),
    }
}
