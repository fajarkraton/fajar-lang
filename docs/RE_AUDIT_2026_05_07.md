## NEW issues surfaced by this re-audit

### NEW-1: Embedded CI failing 5 consecutive runs (HEADLINE) ⚠ HIGH

**Symptom:** all 4 phase17 self-host tests panic with `target/release/fj NotFound` in the Embedded CI Native Codegen Tests job.

**Root cause:** `tests/selfhost_phase17_self_compile.rs::fj_binary()` hardcodes `target/release/fj`; CI runs `cargo test --features native` (dev build) without preceding `cargo build --release --bin fj`.

**Fix candidates (no auto-decision):**

1. **A. Add release-binary build step to embedded.yml** — minimal change:
   ```yaml
   - name: Build release fj binary (required by phase17 tests)
     run: cargo build --release --bin fj
   - name: Run native codegen tests (JIT + AOT)
     run: cargo test --features native
   ```
   Cost: ~3 min added to Embedded CI runtime.
2. **B. Make `fj_binary()` fall back to debug** if release missing —
   alters test semantics (debug interpretive performance is much
   slower; phase17 stage2 native triple-test specifically tests the
   release-binary path so this fix may break the test's intent).
3. **C. Gate phase17 tests behind `#[cfg(release_binary_exists)]`** —
   skip silently if binary missing. Hides the issue, not honest.
4. **D. Make phase17 tests build the binary themselves via
   `Command::new("cargo")...build...`** — slower per-test but self-contained.

**Recommendation:** A. Cleanest, smallest change, preserves test intent.

### NEW-2: Main CI Code Coverage timeout ⚠ MEDIUM

**Symptom:** cargo-tarpaulin run takes >3h35m, timing out the job.

**Probable cause:** running selfhost_phase17_self_compile (the
multi-stage chain) under tarpaulin instrumentation amplifies runtime
massively. Phase 17 tests added in v34.5.8+ likely pushed coverage
runs past the budget.

**Fix candidates:**

1. **A. Exclude self-host phase17 tests from coverage**
   (`--exclude-tests "selfhost_phase17_*"`) — skips coverage of the
   most expensive chain but keeps phase17 itself green.
2. **B. Run coverage only on lib + integ excluding selfhost** —
   coverage is mostly compiler-side, selfhost is integration anyway.
3. **C. Increase timeout / move to nightly schedule** — punt.

**Recommendation:** B. Coverage's purpose is the lib code; selfhost
chain is integration.

### NEW-3: Main CI Windows nightly clippy --tests failure ⚠ LOW

**Symptom:** "Clippy lints (tests)" step failed on
windows-latest+nightly matrix. Linux+stable passes.

**Likely cause:** nightly clippy adds new lints faster than stable,
triggering -D warnings on previously-accepted code.

**Fix candidates:**

1. **A. Bump clippy version pinning to stable on Windows matrix** —
   matches Linux coverage.
2. **B. `#[allow(clippy::new_lint)]` for the specific case** —
   targeted suppression.
3. **C. Run clippy without -D warnings on nightly Windows** —
   permissive.

**Recommendation:** depends on what specific lint. Investigate before
choosing.
