## 8.2 — New stdlib modules

### `stdlib/parser_ast.fj` (346 LOC, 16 fns)

Walks fj source directly (skipping ws + comments via local helpers,
not via the existing token stream — because we need ident text and
literal values that the [i64] tag stream doesn't carry). Produces a
**flat string-tagged AST** array.

AST encoding:

```
BEGIN_FN <name> <ret_type> BEGIN_BODY <stmts> END_BODY END_FN
BEGIN_LET <name> <expr> END_LET
BEGIN_RET <expr> END_RET
BEGIN_IF <cond_expr> BEGIN_THEN <stmts> END_THEN
                    [BEGIN_ELSE <stmts> END_ELSE] END_IF
BEGIN_EXPR_STMT <expr> END_EXPR_STMT

<expr> := INT <value>
        | IDENT <name>
        | <lhs_atom> <rhs_atom> BINOP <op>     ← postfix
        | BEGIN_CALL <fn_name> <args...> END_CALL
```

Architectural choice: `ParseResult { ast: [str], pos: i64, error: bool }`
struct return — same pattern as `stdlib/analyzer.fj`'s `AnalyzerState`.

### `stdlib/codegen_driver.fj` (200 LOC, 8 fns)

Walks the flat AST and drives existing `stdlib/codegen.fj` API
(`emit_preamble`, `emit_function`, `emit_let`, `emit_return`,
`emit_if/else/endif`, `emit_line`).

Top-level entry: `emit_program(ast: [str]) -> str` returns the
complete C source.

Builtin mapping: `println(...)` → `fj_println_int(...)` (matches the
runtime helper emitted by `emit_preamble`).

Dead-code elimination: `parse_atom` / `parse_expr_emit` recursively
handle nested call args + chained binops via the postfix-encoded
AST.

## 8.4 — Generated C example

For input `fn main() -> i64 { let n = 5; if n > 3 { return 111 } else { return 222 } }`:

```c
// Generated by Fajar Lang self-hosted compiler
#include <stdio.h>
#include <stdlib.h>
#include <stdint.h>
#include <string.h>
#include <stdbool.h>

// Fajar Lang runtime
static void fj_println_int(int64_t v) { printf("%lld\n", (long long)v); }
static void fj_println_str(const char* s) { printf("%s\n", s); }
static void fj_println_float(double v) { printf("%g\n", v); }
static void fj_println_bool(int v) { printf(v ? "true\n" : "false\n"); }

int64_t main(void) {
    int64_t n = 5;
    if ((n > 3)) {
        return 111;
    } else {
        return 222;
    }
}
```

Notice the parens around `(n > 3)` — that's the postfix BINOP encoding
producing safe-by-default parenthesization. No precedence bugs.

## 8.5 — What Stage-1-Full does NOT yet support (future work)

These shapes are PARSED partially or not at all by parse_to_ast; would
need additional pattern handlers:

- Multi-fn programs with cross-fn calls (parser.fj's parse_fn already
  handles params; parser_ast doesn't yet — params skipped via
  `while p < n && substring(p, p+1) != ")"`)
- `while` loops (`while cond { body }`)
- `for` loops
- `match` expressions
- `struct` / `enum` definitions
- Generic functions (Stage-1-Full BY DESIGN excludes generics)
- Closures, async, lifetimes (excluded by Stage-1-Subset definition)
- String literals as expression values
- Boolean literals (true/false)
- Float literals

Adding any one is a small extension (10-50 LOC fj per pattern) — the
architecture supports it.

## 8.6 — Sister Rust compiler relationship

**Unchanged.** v33.4.0's stance applies: `src/{lexer,parser,analyzer,
codegen}/` (Rust) is the production fj-lang compiler; `stdlib/lexer.fj
+ parser.fj + parser_ast.fj + analyzer.fj + codegen.fj +
codegen_driver.fj` is the parallel self-host proof. They coexist;
neither replaces the other. Phase 8 deepens the proof to "ARBITRARY
subset programs," not "all fj-lang programs."
